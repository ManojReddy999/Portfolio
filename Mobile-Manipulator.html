Motion Control of a Mobile Manipulator Robot (YouBot)

Demo GitHub

By Manoj Kumar Reddy Manchala | MAE204 - Final Project

Introduction

When I embarked on the challenge of designing a comprehensive control system for a mobile manipulator robot, I knew I was stepping into an exciting yet complex domain. My project focused on the YouBot—a versatile robot capable of performing pick-and-place operations autonomously. In this blog, I’ll take you through the journey of developing the control architecture, the challenges I faced, and the insights I gained along the way.

Project Overview and Objectives

The primary goal of the project was to enable the YouBot to navigate to a specified location, pick up an object, and place it at a target destination. To achieve this, I designed and implemented software components that simulated the robot’s kinematics, generated reference trajectories for the pick-and-place task, and developed a robust feedback control strategy to ensure smooth and precise execution.

The project was implemented in MATLAB using the Modern Robotics Toolbox, with simulation and visualization handled by CoppeliaSim, which offers a versatile platform for testing robotics applications in a controlled, virtual environment, allowing for the detailed observation and analysis of the robot's performance under various conditions.

Methodology

Kinematics Simulator (NextState Function)

This function predicted the robot’s next state using the Euler integration method based on the current state, velocities, and a defined time step. It provided the fundamental basis for planning the robot’s motion.

Reference Trajectory Generator (Trajectory_Generator Function)

The trajectory generator computed a series of configurations for the robot’s end-effector. It ensured a smooth transition between the initial and final positions, creating a continuous path for the pick-and-place operation.

Feedback Control (FeedbackControl Function)

The control strategy combined feedforward commands with a Proportional-Integral (PI) controller to minimize the error between the robot’s actual path and the reference trajectory. Careful tuning of the controller parameters (Kp and Ki) was critical to achieve effective error correction without causing overshoot or oscillations.

Implementation Details

The project was executed primarily in MATLAB, utilizing the Modern Robotics Toolbox for robot kinematics. Simulation and visualization were carried out in CoppeliaSim, with CSV-based communication linking MATLAB-generated trajectories to the virtual environment.

Challenges and Struggles

Tuning the Controller

One of the toughest challenges was tuning the feedback controller. Achieving the optimal balance between the proportional (Kp) and integral (Ki) gains was crucial. I encountered issues like overshoot, slow convergence, and oscillations, all of which required careful adjustments.

Hard-Coding vs. Adaptability

Rather than relying on hard-coded values—which would only work for one set of conditions—I developed a fully adaptable system that could handle changes in initial and final configurations without losing performance.

Handling Singularities

Managing singularities during the calculation of the Jacobian inverse was another significant challenge. Implementing a pseudoinverse with a tolerance helped prevent instability near singular configurations.

Results and Observations

Multiple test cases were evaluated & in all the cases, the initial configuration of the end-effector has at least 30 degrees of orientation error & 0.2m of position error from the first configuration on the reference trajectory:

  • Best Case:

    • An optimized PI controller ensured smooth trajectory tracking with rapid error convergence.

  • Overshoot Case:

    • A less-tuned controller led to overshoots, highlighting the importance of proper tuning.

  • New Task Adaptability:

    • The system successfully adapted to changes in task parameters, including a 90-degree orientation shift.

  • Slow Convergence and High Oscillations:

    • These cases underscored the trade-offs between responsiveness and stability.

Detailed results were documented with error plots and video demonstrations.

Lessons Learned and Future Work

This project deepened my understanding of robot motion control, kinematics, and the challenges of system integration. In the future, I plan to explore advanced control strategies and incorporate real-time sensor feedback to further enhance performance.

Conclusion

The journey of developing a control system for the YouBot was both challenging and rewarding. This project not only refined my technical skills but also offered valuable insights into robotics and autonomous systems. I hope this detailed account inspires further exploration and innovation in the field.
